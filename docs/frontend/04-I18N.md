# 04-I18N.md — Anthekira.dev — Internacionalización

## 1. Propósito

Este documento describe la configuración y uso de **next-intl** para el soporte multilingüe de Anthekira.dev. La Landing Page pública está disponible en **español** (default), **inglés** y **portugués**. El panel administrativo funciona exclusivamente en español.

---

## 2. Configuración de next-intl

### 2.1 Instalación

```bash
npm install next-intl
```

### 2.2 Plugin en `next.config.ts`

```typescript
// next.config.ts
import type { NextConfig } from 'next';
import createNextIntlPlugin from 'next-intl/plugin';

const withNextIntl = createNextIntlPlugin('./src/lib/i18n.ts');

const config: NextConfig = {
  // ... resto de configuración
};

export default withNextIntl(config);
```

### 2.3 Configuración Central — `src/lib/i18n.ts`

```typescript
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing'; // definido abajo

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  // Validar que el locale sea soportado
  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  return {
    locale,
    messages: (await import(`../../public/locales/${locale}.json`)).default,
  };
});
```

> **Nota:** Los mensajes se importan como JSON estático en build time. No hay llamadas a APIs durante el renderizado.

### 2.4 Configuración de Routing — `src/lib/routing.ts`

```typescript
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  // Idiomas soportados
  locales: ['es', 'en', 'pt'],

  // Idioma por defecto (cuando no hay match)
  defaultLocale: 'es',

  // Prefijo de idioma en la URL
  localePrefix: 'as-needed',
  // 'as-needed': / → español (sin prefijo), /en → inglés, /pt → portugués
  // Alternativa: 'always' → /es/, /en/, /pt/ siempre con prefijo

  // Detección de idioma del navegador
  localeDetection: true,
});
```

> **Decisión:** Usar `localePrefix: 'as-needed'` para que `/` sea español (el idioma del propietario) y `/en`, `/pt` sean los otros idiomas. Esto hace que la URL principal sea más limpia para el público objetivo principal (hispanohablantes).

## 4. Routing Localizado

### 4.1 Middleware de next-intl

El middleware se configura en `src/middleware.ts` usando `createMiddleware` de next-intl:

```typescript
// src/middleware.ts
import createMiddleware from 'next-intl/middleware';
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';
import { createServerClient } from '@supabase/ssr';

const i18nMiddleware = createMiddleware({
  locales: ['es', 'en', 'pt'],
  defaultLocale: 'es',
  localeDetection: true,
  localePrefix: 'as-needed', // / → español, /en → inglés
});

export default async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // Rutas admin protegidas (excepto login)
  if (pathname.startsWith('/admin') && !pathname.startsWith('/admin/login')) {
    const supabase = createServerClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
      { cookies: { get: (name) => request.cookies.get(name)?.value } }
    );
    const { data: { session } } = await supabase.auth.getSession();
    if (!session) {
      return NextResponse.redirect(new URL('/admin/login', request.url));
    }
  }

  // Aplicar i18n middleware a rutas públicas
  if (!pathname.startsWith('/admin') && !pathname.startsWith('/api/')) {
    return i18nMiddleware(request);
  }

  return NextResponse.next();
}

export const config = {
  matcher: [
    '/((?!_next/static|_next/image|favicon.ico|images/|fonts/|locales/).*)',
  ],
};
```

### 4.2 Comportamiento de Redirección

| URL solicitada | Comportamiento |
|---|---|
| `anthekira.dev` | Detecta idioma del navegador → redirige a `/` (es), `/en` o `/pt` |
| `anthekira.dev/en` | Muestra Landing en inglés |
| `anthekira.dev/en/projects` | Muestra proyectos en inglés |
| `anthekira.dev/admin` | Muestra admin en español (sin prefijo de idioma) |
| `anthekira.dev/es/admin` | No existe (el middleware no aplica i18n a rutas admin) |

### 4.3 LanguageSwitcher

```tsx
// src/components/shared/LanguageSwitcher.tsx
'use client';

import { usePathname, useRouter } from 'next/navigation';
import { useLocale } from 'next-intl';
import { useState } from 'react';

const locales = [
  { code: 'es', label: 'Español' },
  { code: 'en', label: 'English' },
  { code: 'pt', label: 'Português' },
];

export default function LanguageSwitcher() {
  const locale = useLocale();
  const pathname = usePathname();
  const router = useRouter();
  const [isOpen, setIsOpen] = useState(false);

  const switchLocale = (newLocale: string) => {
    // Reemplazar el prefijo de idioma en la ruta actual
    const newPath = pathname.replace(/^\/(es|en|pt)/, `/${newLocale}`);
    router.push(newPath);
    setIsOpen(false);
  };

  return (
    <div className="relative">
      <button
        onClick={() => setIsOpen(!isOpen)}
        className="flex items-center gap-1 px-3 py-1.5 rounded-md
                   text-sm text-surface-300 hover:text-surface-100
                   hover:bg-surface-800 transition-colors"
      >
        {locale.toUpperCase()}
        <span className="text-xs">▼</span>
      </button>

      {isOpen && (
        <div className="absolute right-0 mt-1 w-36 py-1
                        bg-surface-800 border border-surface-700
                        rounded-lg shadow-xl z-50">
          {locales.map((l) => (
            <button
              key={l.code}
              onClick={() => switchLocale(l.code)}
              className={`w-full text-left px-4 py-2 text-sm transition-colors
                ${locale === l.code
                  ? 'text-primary-600 bg-primary-600/10'
                  : 'text-surface-300 hover:bg-surface-700 hover:text-surface-100'
                }`}
            >
              {l.label}
            </button>
          ))}
        </div>
      )}
    </div>
  );
}
```

```tsx
// src/app/[locale]/projects/page.tsx
import { getTranslations } from 'next-intl/server';
import type { Metadata } from 'next';

export async function generateMetadata({
  params: { locale },

  };
}
```

## 7. Contenido Dinámico del CMS — Auto-traducción con DeepL

### 7.1 Arquitectura

El contenido gestionado desde el CMS (proyectos, servicios, biografía) se traduce **automáticamente** al guardar, no en cada visita.

```
Admin escribe en español
    → Guarda en BD (tabla principal, idioma fuente = ES)
        → Backend detecta campos traducibles
            → Envía texto a DeepL API (www.deepl.com)
                → Recibe traducciones EN y PT
                    → Almacena en tablas de traducción
                        → Landing Page consulta traducciones desde BD
```

### 7.2 Tablas de Traducción

Las traducciones se almacenan en tablas independientes:

```sql
-- Ejemplo para proyectos
CREATE TABLE project_translations (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id  UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  locale      TEXT NOT NULL CHECK (locale IN ('es', 'en', 'pt')),
  title       TEXT NOT NULL,
  description TEXT NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT now(),
  updated_at  TIMESTAMPTZ DEFAULT now(),
  UNIQUE(project_id, locale)
);
```

### 7.3 Consulta desde Server Component

```tsx
// Ejemplo: Server Component que obtiene proyectos traducidos
export default async function ProjectsPage({
  params: { locale },
}: {
  params: { locale: string };
}) {
  const supabase = createServerClient(...);

  const { data: projects } = await supabase
    .from('projects')
    .select(`
      id, slug, image_url,
      project_translations!inner(title, description)
    `)
    .eq('project_translations.locale', locale)
    .eq('active', true);

  return (
    <section>
      {projects?.map(project => (
        <ProjectCard
          key={project.id}
          title={project.project_translations.title}
          description={project.project_translations.description}
          image={project.image_url}
          slug={project.slug}
        />
      ))}
    </section>
  );
}
```

```env
DEEPL_API_KEY=your-deepl-api-key-here
```

## 8. Resumen de Estrategias de Traducción

| Tipo de contenido | Método de traducción | Almacenamiento |
|---|---|---|
| **UI (menús, botones, etiquetas)** | Archivos JSON con next-intl | `public/locales/*.json` |
| **SEO (metadatos, títulos de página)** | Archivos JSON con next-intl | `public/locales/*.json` |
| **Contenido dinámico (proyectos, servicios, bio)** | Auto-traducción con DeepL al guardar | Tablas de traducción en PostgreSQL |
| **Panel admin** | Solo español (sin traducción) | — |
